## ğŸ’» Installation & Setup

### 1. Prepare the dataset
The application expects the CSV files under the **topâ€‘level `data/` folder**. By default the repository contains two files:

- `data/dataset.csv` â€“ the raw Spotify export
- `data/cleaned_dataset.csv` â€“ generated by running the cleaner script

If `cleaned_dataset.csv` is missing you can create it with:

```bash

```

This step is performed automatically by `run_services.py` (see below).

### 2. Backend Setup
```bash
cd backend
python -m venv venv
# Windows
venv\Scripts\activate
# Linux/Mac
source venv/bin/activate

pip install -r requirements.txt
```

#### Running the API
There are two supported modes:

1. **Monolithic** (all endpoints in a single app):
   ```bash
   python main.py          # serves at http://localhost:8000
   ```

2. **Microservices** (split into auth/recommender/analytics/playlist):
   the helper script will start all four and generate the dataset if needed:
   ```bash
   python run_services.py  # ports 8001â€“8004
   ```

The frontend chooses which URLs to call based on environment variables; by default it will hit the monolith at `http://localhost:8000` unless you set `REACT_APP_AUTH_SERVICE`, `REACT_APP_RECOMMENDER_SERVICE`, etc.

### 3. Frontend Setup
```bash
cd frontend
npm install
npm run dev
```

### 4. Running with Docker (optional)
The repository includes Dockerfiles for each microâ€‘service and a `docker-compose.yml` that spins up all
services plus a PostgreSQL container.

Before starting the stack you need to build the shared base image and make sure the
`data/` folder (containing `cleaned_dataset.csv`) and `.env` file are available. The compose file mounts
them into the containers.

```bash
# build base image used by all services
docker build -t spotify-base -f backend/Dockerfile.base backend

# start the whole architecture
docker-compose up --build
```

The `docker-compose.yml` also supports an `env_file` entry, so your `backend/.env` will be loaded
automatically; you can still override individual variables via the usual `-e` flags.

Once the containers are running the frontend will be served on http://localhost (port 80) and the
APIs on ports 8001â€“8004.  Be sure the dataset is present or the recommender will still return
empty results despite the services being up.

To stop the stack press **Ctrlâ€‘C** (or `docker-compose down`).
